# AI Fact-Checkers in the Wild — Replication Code

Code and data for replicating the analyses in *"AI Fact-Checkers in the Wild: A Field Study of LLM-Written Community Notes on X."*

This repository compares LLM-written Community Notes against human-written Community Notes on the same tweets, covering note-level outcomes, rating-level ideology patterns, text/source analysis, and robustness checks.

## Directory Structure

```
code/
├── analysis.py              # All paper analyses (Python)
├── process_data.py          # Data loading and preprocessing pipeline
├── rating_analysis.R        # Rating-level mixed-effects models and subgroup HTE (R)
├── data/
│   ├── all_notes.csv            # Pre-computed bot + human notes
│   ├── filtered_ratings.csv     # Pre-computed filtered ratings
│   ├── ratings_analysis_df.csv  # Rating-level analysis dataset (for R and Python)
│   ├── complete_rater_ratings.csv  # Equal-exposure (complete rater) subset
│   ├── human_crh_hit_rate.csv   # Per-author CRH/hit-rate stats for human writers
│   ├── noteParams.tsv           # Note parameters from complete-rater scoring
│   ├── tweet_subgroups.csv      # Topic and multimodal labels per tweet
│   ├── tids_api_retrieved.txt   # Tweet IDs targeted by bot accounts
│   └── api_account_ids.json     # Bot account participant IDs
└── outputs/
    ├── analysis_report.md
    ├── analysis_report_complete_raters.md
    ├── rating_analysis_bot_vs_human_barchart.png
    ├── rater_distribution_full_vs_complete_raters.png
    ├── pairwise_bt_comparisons.csv
    ├── hte_topic.png            # (from R)
    └── hte_modality.png         # (from R)
```

## Data Files

| File | Description |
|------|-------------|
| `all_notes.csv` | Combined bot + human notes with columns: `noteId`, `noteAuthorParticipantId`, `createdAtMillis`, `tweetId`, `summary`, `isMediaNote`, `classification`, `writer`, `finalRatingStatus`, `numRatings`, `coreNoteIntercept`, etc. |
| `filtered_ratings.csv` | Ratings filtered to analysis notes and raters with valid helpfulness factors (`coreRaterFactor1`, `coreRaterIntercept`) |
| `ratings_analysis_df.csv` | Merged ratings with note writer labels, `AI` indicator, and `rating_score` — used by both `analysis.py` and `rating_analysis.R` |
| `complete_rater_ratings.csv` | Equal-exposure subset: ratings from raters who rated every note on a tweet |
| `human_crh_hit_rate.csv` | Per human author: `n_total`, `n_crh`, `n_crnh` (for CRH percentile analysis) |
| `noteParams.tsv` | Note-level parameters (`internalNoteIntercept`, `internalNoteFactor1`, `numRatings`) from complete-rater Community Notes scoring |
| `tweet_subgroups.csv` | Per-tweet labels: `topic` and `multimodal` (used by `rating_analysis.R` for HTE) |
| `tids_api_retrieved.txt` | Tweet IDs targeted by bot accounts (one per line) |
| `api_account_ids.json` | Mapping of bot account labels to participant ID strings |

The processed data files in `data/` were generated by running `process_data.py` on raw Community Notes data (https://communitynotes.x.com/guide/en/under-the-hood/download-data) downloaded on 02/03/2026. Since the outputs are already saved, you can directly run `analysis.py` and `rating_analysis.R` to replicate all analysis results without re-downloading the raw data.

## Usage

### 1. Data Processing

```bash
# Load pre-computed CSVs (fast-start, default)
python process_data.py

# Process from raw Community Notes TSVs (requires raw data download)
python process_data.py --no-fast-start

# Build complete-rater ratings (equal-exposure subset)
python process_data.py --complete-raters
```

### 2. Analysis (Python)

```bash
# Run all analyses (full sample)
python analysis.py

# Run specific analyses
python analysis.py --analysis rating note timing text crh timing_matched pairwise_bt

# Equal-exposure analyses 
python analysis.py --analyze-with-complete-raters

# Rater distribution comparison (Appendix D)
python analysis.py --rater-distribution

# Pairwise Bradley-Terry analysis (Appendix A)
python analysis.py --analysis pairwise_bt
```

### 3. Analysis (R)

Run `rating_analysis.R` in an R session after the Python data processing step has produced `data/ratings_analysis_df.csv` and `data/tweet_subgroups.csv`.

Note: the script uses `setwd("~/cn-ai-writer/code")` — adjust this path or run from the `code/` directory.
